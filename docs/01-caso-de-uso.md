# Caso de uso - Mini ERP-cRM en Odoo

## Contexto
Este proyecto simula la implantación de un ERP-CRM en una organización de servicios / asociación, con el objetivo de gestionar de forma centralizada:
- clientes
- oportunidades
- actividades
- servicios prestados
- facturación

El sistema se basa en Odoo Community, configurado y extendido de forma básica.

## Objetivo. Disponer de una herramienta que permita:
- Registrar y mantener datos de clientes de forma estructurada.
- Realizar el seguimiento comercial desde el primer contacto hasta la prestación del servicio.
- Obtener indormación clara para la toma de decisiones.

## Usuarios del sistema
- Administrador: configura el sistema, usuario y módulos.
- Gestor comercial / CRM: registra clientes, oportunidades y actividades.
- Gestor administrativo: supervisa servicios y facturación (simulada).

## Alcance
Este proyecto cubrirá toda la complejidad de un ERP real, sino desmostrar:
- Comprensión de procesos ERP-CRM
- Capacidad de configuración funcional
- Uso de datos y reporting básico.
